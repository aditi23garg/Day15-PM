## 🧠 Concepts Covered

### 🔹 1. ER Diagram (Entity Relationship Model)

* Designing database for **online food delivery app**
* Entities: Customer, Restaurant, Order, Menu Item, Delivery Agent
* Understanding relationships (1:N)

---

### 🔹 2. Database Normalization (1NF → 2NF → 3NF)

* Removing redundancy
* Avoiding anomalies (update, insert, delete)
* Converting flat table into structured schema

---

### 🔹 3. Relational Algebra

* Selection (σ) → filtering data
* Projection (π) → selecting columns
* Join (⨝) → combining tables
* Set operations (Union, Difference)

---

### 🔹 4. Mapping SQL & Pandas to Relational Algebra

* Understanding how:

  * SQL queries
  * Pandas operations
    map to mathematical database operations

---

### 🔹 5. BCNF (Boyce-Codd Normal Form)

* Advanced normalization concept
* Removing dependency issues
* When to use 3NF vs BCNF

---

### 🔹 6. Database Anomalies

* Update anomaly
* Insert anomaly
* Delete anomaly
* Understanding why normalization is important

---

### 🔹 7. ACID Properties

* Atomicity
* Consistency
* Isolation
* Durability
* Example: preventing double booking in systems

---

### 🔹 8. Real-World Database Design

* Food delivery system
* Price history tracking
* Ride-sharing schema design
